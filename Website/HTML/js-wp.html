Javascript: Front-end

JavaScript is a high-level, web-scripting programming language that is widely used for making the websites more interactive.

Variable Declaration

In Javascript, variable declaration is done by writing var and then a variable name, then assigning to it a value. The value can be any datatype; it is also possible to not assigned a declared variable a value.

Example:

 
var a = 0;
var b;
var c = "Hello World"
 

Data Types

Number

The datatype number can be a decimal integer, decimal floating point or a decimal scientific notation. It can used to represent numbers in base 2, 8 or 16. There are also special numbers, namely infinity, negative infinity and NaN. Due to the nature of Javascript, division with numbers will always result as a decimal floating point and beware of inexact result.

Example:

 
var num = 10;
var number = 1.5;
var numbers = 3.1e7;
 

String

The string datatype can be represented by a single quote ('') or a double qoute ("").

Example:

 
var str = "Hello World";
 

Boolean

The boolean data type has two values. The first is the falsy values which is represented by false, null, 0, '', and undefined. The second is the truthy values which are the values other than the falsy values.

Example:

 
var condition = true;
var condition = 0;
var condition = null;
 

Null or Undefined

These data types are the default when a variable is declared and is not assigned to any value(eg var none;, var none = null;).

 
var none = null;
var none;
 

Object

Object types corresponds to a combination primitive data types, such as number and boolean, used to represent data.

 
var obj = {title: "Test", test: "Hello World", val: 13};

Example Function declaration and Invocation:

if(a>b){
return a;
}else{
return b;
}
}

fnc("Hello", "World");
fnc(3, true);
 

There is an object called argument in javascript which is an array that treats the parameter of a function as a data inside the array

 
function fnc1(a,b){
console.log(argument[0]); //the output is the value of a

Arrays

Arrays in java script are dynamic in size and it allows sparse arrays.

Methods used in Arrays

push: adds a net item at the end of the array.

 
var colors = ["red", "blue", "green"];
colors.push("yellow");
 
<- colors = ["red", "blue", "green", "yellow"]
 

pop: removes the item at the end of the array.

 
var colors = ["red", "blue", "green"];
colors.pop();
 
<- colors = ["red", "blue"]
 

unshift: adds an item at the beginning of the array.

 
var colors = ["red", "blue", "green"];
colors.unshift("yellow");
 
<- colors = ["yellow", "red", "blue", "green"]
 

shift: removes the item at the beginning of the array.

 
var colors = ["red", "blue", "green"];
colors.shift();
 
<- colors = ["blue", "green"]
 

sort: sorts the elemets of the array in lexicographic order.

 
var colors = ["red", "blue", "green"];
colors.sort();
 
<- colors = ["blue", "green", "red"]
 

reverse: reverses the elemts of the array.

 
var colors = ["red", "blue", "green"];
colors.push("yellow");
 
<- colors = ["green", "blue", "red"]
 

Other and more complex methods:

 

forEach: calls the elements of the array one at a time.

 

reduce: reduces the the value of the array to single value.

 

filter: filters the elements of the array that passes the condition

 

map: maps a function once for each elements of the array.

Event Handling

Event Propagation

Event propagation determines in which order the elements receive the event. There are two ways of event propagation namely Capture phase and Bubbling phase.

 

Capture Phase

with capturing, the first event to be captured and handled is the outermost elememt going to the innermost element.

 
<div>
<ul>
<li><li>
<ul>
<div>
 

In this example, assume that a click event happened on <li>, the order of propagation would then be from <div> to <li>

 

Bubbling Phase

with bubbling, the first event to be captured and handled is the innermost elememt going to the outermost element..

 
<div>
<ul>
<li><li>
<ul>
<div>
 

In this example, assume that a click event happened on <li>, the order of propagation would then be from <li> to <ul> to <div>

Execution of Synchronous and Asynchronous Codes

JavaScript is a single-threaded programming language which means that it can only execute one statement at a time.

 

Synchronous

this means that while a statement is being executed, following line of statements can't be executed.

 
cosole.log("First");
cosole.log("Second");
cosole.log("Third");
 

In this example, when the statement console.log(" First "); is executed, following statements are blocked and can not be executed.

 

The output should then be:

 
First
Second
Third
 

Assynchronous

This means that while a statement is being executed, the next line of codes can be executed even though current statement are executing. It may seem that a synchronous execution is straightforward but it can really be slow. With asynchronous execution, things can get faster especially with image processing, file operation and others.

 
console.log(1);
setTimeout( function() {
consoleLog.(3);
});
cosole.log(2);
 

In this exmaple, we used setTimeout(), let us say that the parameters for the function is setTimeout(callback, 2000).It has the component callback and 2000 ms, callback will let the user know if the execution is done within the time of 2 seconds. The first statement is executed then the second one which is the function setTimeout(). While it is executing, lines after it can be executed. After 2 seconds, callback is now triggered and the statement is now executed.

 

The output should then be:

 
1
2
3

Promise

Promise Objects may produce a single value some time in the future that can be either a resolve value or a rejected value. A promise can have 3 states mainly: fulfilled, pending, or rejected. Promises are eager, that means that a promise will start doing task you give it as soon as the promise constructor is invoked.

 

3 States:

  • If the state is fulfilled, onFulfilled() will be called (e.g. resolve())
  • If the state is rejected, onRejected() will be called (e.g. reject())
  • If the state is pending, it is notyet fulfilled or rejected
