The World Wide Web

The World Wide Web or WWW is a collection of web resources, digital repository or storage of data, and applications which allows access of the web resources through the internet. The WWW was invented by Sir Tim Bernes-Lee back in March 1989, its initial purpose was to provide a way for researchers to share and information within a wide range.

Types of Web Resources

  • Static

    • Static resources are reources that are exaclty stored and do not require web programming and database design.

  • Dynamic

    • Dynamic resoursec are resources that are generated, usually Client-side-scripting, server-side scripting, or both are used



Types of Applications

  • Client Application

    • An application that can acquire or secure the services of other programs usually server appplications.

  • Server Application

    • An application that waits for requests from other applications, whether client or server application, and sends back a response accordingly

Universal Resource (L|I|N)

Different web pages are stored or kept in web servers around the world and upon clicking a link or inputing an URL/URI/URN of a webpage triggers a request for the web page from the web server, the server then transmits back a response containing the web page data that your web browser displays.

Universal Resoure Identifier

A means of identifying any kind of web resource whether it be text, image and etc. It divides into two types, namely URL and URN.



Universal Resoure Locator

Indicates where a web resource is located in the internet. Another way of describing an URL is that it is a reference to a web resource stored in the internet.

Parts:

scheme ://authority(deprecated) /path :port number /query /fragment identifier(optional)

(i.e. https://www.google.com.ph/database:8080/search?q=test#first)



Universal Resoure Name

The URN is a resource reference that is similar to the URL but instead of the location of the resource, a specific name is assigned to the resource making it easy for a person or program to find the resource. The URN is also a type of a URI.

Hyper Text Transfer Protocol

HTTP is an application layer communication protocol that is used to access resources(hypertext/hypermedia) from the World Wide Web. Invented by Tim Berners Lee at CERN(Conseil Européen pour la Recherche Nucléaire) in 1989 together with W3C(World Wide Web Consortium) and IETF(Internet Engineering Task Force).



Versions

  • HTTP 0.9: (1991)-(TCP/IP connection using port 80; "GET"; only html)
  • HTTP 1.0: (RFC(Remote Function Call) 1945, May 1996)-(pull protocol; always initiated by client)
  • HTTP 1.1: (RFC 2068 January 1997, RFC 2616 June 1999); RFC 7230-7235 June 2014
  • HTTP 2.0: (RFC 7540 May 2015)-(pipelining introduced; multiplexing; server push - server taking initiative)


Fundamentals

HTTP runs on top of TCP(Transmission Control Protocol/IP(Internet Protocol) with the use of TCP port 80 by default or TCP port 443 for HTTPS(Secure HTTP)

  • HTTP is based on a client-server architecture
  • HTTP uses a request-response standard protocol
  • HTTP is a stateless communications protocol
  • HTTP Functionalities: cache control, content media type specification, content/transfer coding, content negotiation, client-server protocol negotiations, persistent connections, request pipelining/ multiplexing (HTTP2) and authorization/authentication

Request Message

When a client interacts with a server it sends out a request message, this request message contains various components and they are formatted as demonstrated below.


  • Request line (CRLF - terminated line consisting of three space separated values)
    • Method
    • Request Target
    • Protocol Version
  • Message Header
  • Empty Line
  • Pay Load


Message Header

  • General Header Fields: entity header fields
  • Request Header Fields: specific information about the resource to be retrieved
  • Response Header Fields: additional information about the response
  • Entity Header Fields(Payload): information about the body of the entity

Request Methods

Request methods are a way to indicate what type of interaction the client would do to a requested resource or resources. Below are the standard request methods of HTTP.


  • GET
    • transfer a current selected representation of the target resource.
  • HEAD
    • retrieves the status line and/or headers, similar to GET but without the payload.
  • POST
    • request that the target resource perform resource-specific processing of the representation enclosed in the message body of the request. Typically used in submitting HTML form data.
  • PUT
    • create or replace the state of the target resource with the state defined by the representation enclosed.
  • DELETE
    • removes the association between the target resource and its current functionality.
  • OPTIONS
    • request information about the communication options available for the target resource.
    • the request target may be "*".
  • TRACE
    • request remote, application level loopback of the request message(request the recipient echo back to the client received request message).
  • CONNECT
    • request establishment of a tunnel to the destination origin server and if succesful thereafter restricts its behavior to blind-formating of products in both direction.

Method Properties

Safe Methods

Methods that can only perfom "read-only" procedures.

e.g. GET, HEAD, TRACE and OPTIONS



Idempotent Methods

Methods that are read-only or not and can only be effective once.

e.g. GET, HEAD, TRACE, OPTIONS, PUT and DELETE



Cacheable Methods

Methods that allows to be stored for future reference.

e.g. GET, HEAD and POST
